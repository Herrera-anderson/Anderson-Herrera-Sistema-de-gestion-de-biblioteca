¿Qué es Markdown y por qué se utiliza en proyectos de software?
Markdown es un lenguaje de marcado ligero que permite formatear texto usando caracteres sencillos en lugar de etiquetas complejas como HTML. Se utiliza en proyectos de software porque su simplicidad, legibilidad y facilidad de uso agilizan la creación de documentación, como archivos readme.md
Se utiliza en proyectos de software por lo simple y fácil que es modificarlo de forma que se hace increíblemente intuitiva, incluso en texto plano, haciendo que sea legible para cualquiera, además de que paginas como github muestran el .readme como descripción para las paginas de proyectos.
# Ventajas de usar Markdown en GitHub

| Ventaja                                  | Descripción                                                                 |
|-------------------------------------------|-----------------------------------------------------------------------------|
| Simplicidad                               | Es fácil de escribir, leer y editar sin herramientas avanzadas.            |
| Compatibilidad con GitHub                 | GitHub renderiza Markdown de forma automática en README y documentación.   |
| Control de versiones                      | Al ser texto plano, los cambios se rastrean fácilmente en Github.          |
| Portabilidad                              | Los archivos .md funcionan en cualquier editor o sistema operativo.        |
